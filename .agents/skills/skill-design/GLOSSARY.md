# Glossary — Skill Design

Ключевые термины для проектирования скиллов. Используется как disclosed reference из SKILL.md.

## Predictability

Степень, с которой скилл заставляет агента вести себя одинаковым *образом* на каждом запуске — одинаковый процесс, не одинаковый вывод. Корневая добродетель, которой служат все остальные термины.

## Invocation

### Model-Invoked

Скилл, у которого есть description — агент видит его и может запустить автономно. Платит постоянный Context Load в каждом тёрне. Reachable by other skills.

### User-Invoked

Скилл без description — невидим для агента, доступен только человеку. Нулевой Context Load, но платит Cognitive Load (человек должен помнить о скилле).

### Description

Поле YAML frontmatter, которое одновременно является единственным Context Pointer'ом model-invoked скилла. Его присутствие — это и есть ось инвокации.

### Context Load

Стоимость, которую model-invoked скилл налагает на контекстное окно агента — его description загружен всегда.

### Cognitive Load

Стоимость, которую user-invoked скилл налагает на человека — нужно помнить, какие скиллы существуют и когда их вызывать.

### Router Skill

User-invoked скилл, который перечисляет другие user-invoked скиллы и подсказывает, когда какой использовать. Лекарство от Cognitive Load.

## Information Hierarchy

### Information Hierarchy

Контент скилла, ранжированный по тому, насколько срочно он нужен агенту:

1. **Steps** — в SKILL.md, первичные
2. **Reference** — в SKILL.md, вторичные
3. **Reference, disclosed** — за Context Pointer

### Steps

Упорядоченные действия, которые агент выполняет. Каждый шаг заканчивается Completion Criterion.

### Reference

Материал, к которому агент обращается по мере необходимости: определения, факты, параметры, примеры.

### External Reference

Reference за пределами скилла — обычный файл, без description, не invocable.

### Progressive Disclosure

Перемещение Reference вниз по иерархии — из SKILL.md за Context Pointer — чтобы верх оставался читаемым.

### Context Pointer

Ссылка в контексте агента, которая именует материал вне контекста и кодирует условие для его загрузки. Его формулировка, а не цель, решает, *когда* и *насколько надёжно* агент до него дойдёт.

### Co-location

Размещение материала, который агенту нужен одновременно, в одном месте.

### Sprawl

*Failure mode.* Скилл, который просто слишком длинный — независимо от причины.

## Steering

### Branch

Отдельный способ вызова скилла — один из нескольких путей через скилл.

### Leading Word

Компактное понятие из претренировки модели (Leitwort), которым агент мыслит, выполняя скилл. Якоряет предсказуемость минимальным числом токенов.

### Completion Criterion

Условие, которое говорит агенту, что работа над шагом завершена. Два свойства: clarity (отличимо done от not-done) и demand (насколько тщательную работу требует).

### Legwork

Работа, которую агент делает за кулисами внутри одного шага: чтение файлов, исследование кодовой базы. Живёт ниже структуры шагов.

### Post-Completion Steps

Шаги, следующие за текущим. Видимые, они тянут агента вперёд к Premature Completion.

### Premature Completion

*Failure mode.* Завершение шага до того, как он действительно сделан — внимание агента соскальзывает на «быть сделанным», а не на работу.

## Pruning

### Single Source of Truth

Желаемое состояние, где каждое значение живёт ровно в одном авторитетном месте.

### Duplication

*Failure mode.* Одно и то же значение в более чем одном месте.

### Relevance

Относится ли строка к тому, что делает скилл — линза для решения, что оставить.

### Sediment

*Failure mode.* Слои старого контента, которые оседают в скилле и никогда не чистятся.

### No-Op

*Failure mode.* Инструкция, которая ничего не меняет, потому что модель уже делает это по умолчанию.

## Skill Types

### Discipline Skill

Скилл, навязывающий дисциплину: правила и требования, которые агент склонен нарушать под давлением. Требует bulletproofing (rationalization table, red flags, prohibition).

### Technique Skill

Конкретный метод с шагами для выполнения: condition-based-waiting, root-cause-tracing.

### Pattern Skill

Способ мышления о проблемах: flatten-with-flags, test-invariants.

### Reference Skill

API-документация, синтаксические справочники, описание инструментов.
