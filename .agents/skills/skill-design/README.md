# skill-design

Проектирует спецификации для AI-скиллов: от идеи до готового документа с проверкой качества и тестовой стратегией.

## Когда использовать

- Создать новый скилл с нуля (greenfield)
- Доработать существующий скилл (iteration)
- Получить ревью и проверку качества существующей спецификации

## Как это работает

Скилл проходит 5 фаз в гейтированном режиме:

```
PHASE 0: Setup       — определение типа скилла и режима работы
PHASE 1: Probe       — интервью (grilling-style, одна ветка за раз)
PHASE 2: Spec        — написание спецификации по шаблону
PHASE 3: Quality     — проверка качества (No-Op, leading words, placeholders)
PHASE 4: Test        — тестовая стратегия для будущего скилла
```

Переход к следующей фазе только после верификации текущей.

## Структура

```
.agents/skills/skill-design/
├── SKILL.md              # Основной файл — процесс проектирования (загружается моделью)
├── README.md             # Эта документация
├── GLOSSARY.md           # Глоссарий терминов (Context Load, Leading Word, Completion Criterion и др.)
├── spec-template.md      # Шаблон выходного документа — спецификации скилла
└── review-checklist.md   # Чеклист проверки качества для Фазы 3
```

## Режимы

### Greenfield (создание с нуля)
Полный цикл из 5 фаз. Используется, когда SKILL.md ещё не существует.

### Iteration (доработка)
Сокращённый цикл — Фаза 0 загружает существующий SKILL.md и проводит дельта-анализ:
- Что оставить (работает)
- Что изменить (недостаточно)
- Что добавить (новые требования)
- Что удалить (устарело)

Режим определяется автоматически: если SKILL.md существует → iteration.

## Выходной артефакт

Спецификация сохраняется в `docs/skills-specs/<skill-name>-spec.md` и содержит:
- YAML frontmatter (name, SDO-оптимизированное description)
- Структуру секций
- Information hierarchy (что в SKILL.md, что в supporting files)
- Completion criteria и Verification checklist
- Common Rationalizations и Red Flags
- Тестовую стратегию

В режиме iteration — дельта-спецификация (keep/modify/add/remove).

## Зависимости

Этот скилл не требует внешних инструментов. Все проверки выполняются inline.
